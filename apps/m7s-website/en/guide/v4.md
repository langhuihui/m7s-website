# v4 Release Notes
## Main changes from V3 to V4
- Refactored core structs using generics.
- RingBuffer now stores complete frames to ensure accurate marker insertion when constructing RTP packets.
- Removed GOP delayed frame tracking logic, now using sync-to-latest-keyframe technology.
- Cache changed from []byte to net.Buffers for improved performance using writev.
- Pre-built avcc, flv, and rtp formats stored in cache for better memory reuse.
- Support for dynamic bitrate and resolution.
- Stream now has 4 states, enabling complex scenarios like on-demand playback and reconnection after disconnection.
- Removed Track waiting logic, now using event notifications.
- Modified Hook mechanism to an event bus model, simplifying code logic.
- Added HTTP callback configuration.
- Updated plugin installation API for easier use.
- Plugins automatically register Web APIs through reflection and provide middleware mechanisms.
- Includes CORS configuration and basic authentication functionality.
- Updated subscribe/publish API to use a scalable and unified event model.
- Uses zap as the logging library.
- Config file format changed to YAML.
- Can now be launched without a config file (uses default config).
- Plugins must be explicitly disabled in the config file to prevent them from being activated.
- Subscription, Publishing, Pushing, and Pulling configurations are now covered by global, plugin-level, and stream-level priority override mechanisms.
- Basic architecture for remote server pushing and pulling implemented, making it easier for plugins to implement client-side logic.
- Complete support for h265.
- Added debug plugin for debugging purposes.
- Removed Gateway and summary plugins and merged them into the engine.
- Record plugin now supports fmp4 format recording.
- Added webtransport plugin for WebTransport protocol playback.
- RTSP plugin now supports h265 playback and can process AAC and G711 audio streams.
- RTSP protocol forwarding no longer repacks and re-encapsulates packets--now transparent.
- RTMP protocol has implemented client-side logic that allows streams to be pushed or pulled from remote RTMP servers.
